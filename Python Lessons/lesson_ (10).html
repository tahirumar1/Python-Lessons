What this project brings together

This is the first project that feels like a real game. Every concept from Week 2 is woven together: if/elif/else for hints, and/or/not for compound checks, nested conditions for validation, and a while loop with break to keep the game running. Students who finish the core requirements should have something they can genuinely share and play with friends.

🎯Project day goal: Every student leaves with a complete, playable game. Not a tutorial — their own design, their own variable names, their own messages. Circulate early and often; unblock stuck students by asking questions rather than giving code.
Learning objectives
  • Combine while loops, conditionals, and input validation in one complete program
  • Use import random and random.randint() to generate a secret number
  • Implement a play-again outer loop so the game restarts without re-running the file
  • Track and display a best-score (fewest tries) across multiple rounds
  • Present code to a partner and explain at least two design decisions

guessing_game.py — complete teacher solution
import random

print("=" * 40)
print("     NUMBER GUESSING GAME")
print("=" * 40)

MAX_TRIES  = 7
best_score = None    # None means "no score yet"
round_num  = 1

# ── Outer loop: play again ──────────────────────
while True:
    print(f"\nRound {round_num} — Guess a number between 1 and 100.")
    secret = random.randint(1, 100)
    tries  = 0
    won    = False

    # ── Inner loop: guessing ───────────────────────
    while tries < MAX_TRIES:
        remaining = MAX_TRIES - tries
        raw = input(f"  Guess ({remaining} tries left): ")

        # validate input
        if not raw.isdigit():
            print("  Please enter a whole number.")
            continue                       # don't count invalid input

        guess = int(raw)
        tries += 1

        if guess == secret:
            print(f"  🎉 Correct! You got it in {tries} {'try' if tries == 1 else 'tries'}.")
            won = True
            break
        elif guess < secret:
            print("  Too low — try higher. ↑")
        else:
            print("  Too high — try lower. ↓")

    # ── End of round ───────────────────────────────
    if not won:
        print(f"  💀 Out of tries! The number was {secret}.")
    else:
        if best_score is None or tries < best_score:
            best_score = tries
            print("  🏆 New best score!")
        print(f"  Best score so far: {best_score} tries.")

    # ── Play again? ─────────────────────────────────
    again = input("\nPlay again? (yes/no): ").lower().strip()
    if again != "yes":
        print(f"\nThanks for playing! Final best score: {best_score} tries.")
        break
    round_num += 1

Check for understanding — self-assessment exit ticket
In the guessing game, why is best_score initialised to None rather than 0?
ANone and 0 are the same thing in Python — either works
BBecause 0 would crash when compared to an integer try count
CNone means "no score yet" — if we used 0, no real score could ever beat it, since you can't win in 0 tries
DBecause random requires scores to be initialised as None

Teacher notes
🚨Most common structural bug: Students who put the random.randint() call outside the outer loop — meaning the same secret number is used every round. Watch for this during circulation. Ask: "Where does a new secret number need to be picked?"
⚠️Common mistake: The best_score update logic — students who write if tries < best_score without first checking if best_score is None will crash on the first round since you can't compare an int to None. Show them the is None or pattern.
⚠️Common mistake: Students who don't reset tries = 0 and won = False inside the outer loop — so Round 2 inherits Round 1's state. Teach: every variable that belongs to a single round must be re-initialised at the start of each round, inside the outer loop.
💡Pacing: 27 minutes is enough for the core 8 requirements if students planned on paper first. Students who struggle with the play-again loop should skip the best-score tracker for now — a working core game is better than a broken extended one.
❤️Celebration: Week 2 is complete. Students who joined with no coding experience now have two working, interactive programs — a Mad Libs generator and a number guessing game. Say that out loud. Let them feel it before previewing Week 3.
