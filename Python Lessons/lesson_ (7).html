The three logical operators
and
Both must be True
Returns True only when every condition is True. If any one is False — the whole expression is False.
age >= 13 and has_ticket
→ True only if BOTH hold
or
At least one must be True
Returns True when at least one condition is True. Only False when every condition is False.
is_member or has_coupon
→ True if EITHER holds
not
Flips True ↔ False
A unary operator — takes only one condition and inverts it. not True → False. not False → True.
not is_banned
→ True if they are NOT banned

Short-circuit evaluation — Python stops early

Python evaluates logical expressions left to right and stops as soon as the result is determined. This is called short-circuit evaluation. It saves time and prevents errors.

💡Why it matters in practice: Short-circuit prevents crashes. Example: if x != 0 and 10/x > 2 — if x is 0, Python short-circuits and never divides by zero. Students don't need to use this today, but knowing it exists explains confusing behaviour later.
Live demo — code examples
Example 1 — and: both conditions must be true
main.py
age        = int(input("Enter age: "))
has_ticket = input("Do you have a ticket? (yes/no): ")

if age >= 13 and has_ticket == "yes":
    print("Welcome! Enjoy the show.")
else:
    print("Sorry, you cannot enter.")
Four possible outcomes
age=15, ticket=yes → Welcome! Enjoy the show.
age=15, ticket=no → Sorry, you cannot enter.
age=10, ticket=yes → Sorry, you cannot enter.
age=10, ticket=no → Sorry, you cannot enter.
Example 2 — or: at least one condition true
main.py
is_member  = input("Are you a member? (yes/no): ")
has_coupon = input("Do you have a coupon? (yes/no): ")

if is_member == "yes" or has_coupon == "yes":
    print("Discount applied! 20% off.")
else:
    print("Full price applies.")
Outcomes
member=yes, coupon=yes → Discount applied! 20% off.
member=yes, coupon=no → Discount applied! 20% off.
member=no, coupon=yes → Discount applied! 20% off.
member=no, coupon=no → Full price applies.
Example 3 — not: flip a condition
main.py
is_banned = False
username  = input("Enter username: ")

if not is_banned:
    print(f"Welcome, {username}!")
else:
    print("Access denied.")

# not also works inline with comparisons
score = 45
if not score >= 50:        # same as: score < 50
    print("Did not pass.")
Example 4 — Combining all three + common mistake
main.py
temp = int(input("Temperature in °C: "))

# ✓ Correct — checks temp is between 20 and 30
if temp >= 20 and temp <= 30:
    print("Perfect weather!")

# ✗ Common mistake — this ALWAYS passes (reads wrong)
if temp >= 20 and <= 30:     # SyntaxError!
    print("Wrong syntax")

# Python also allows chained comparisons (very readable!)
if 20 <= temp <= 30:          # Pythonic shorthand
    print("Perfect weather! (chained)")
💡The range-check mistake is universal. Every beginner writes 20 <= temp <= 30 in their first language and it works. When they write temp >= 20 and <= 30 in Python (missing the variable name on the right side), it crashes. Show this live, let them see the SyntaxError, then show the fix. Also reveal the chained comparison — it will delight them.
Activity 1 — Movie night checker
Activity 1 · 15 minutes · Individual
Build a cinema access checker using and, or, and not
A cinema has a PG-13 film and an R-18 film showing. Students write a program that checks the viewer's age and parental permission, then decides which film (if any) they can watch — using all three logical operators.
  1. Ask for the viewer's age (int) and whether they have parental permission (yes/no)
  2. R-18 film: age must be 18+ OR (age 13–17 AND has permission) — use and + or
  3. PG-13 film: age must be 13+ OR has permission — simpler or condition
  4. Use not in at least one branch (e.g. for the "no access" case)
  5. Extension: add a third film (G-rated, any age) and a fourth output "no films available"
Starter code
main.py — starter
print("=== Cinema Access Checker ===")
age        = int(input("Your age: "))
permission = input("Parental permission? (yes/no): ")

# R-18: must be 18+ OR (13-17 with permission)
if age >= 18 or (age >= 13 and permission == "yes"):
    print("✓ You can watch the R-18 film.")
# PG-13: must be 13+ OR has permission
elif ???:
    print("✓ You can watch the PG-13 film.")
else:
    print(f"✗ Sorry, no films available for age {age}.")
Example test cases — run all four to verify
age=20, permission=no → ✓ You can watch the R-18 film.
age=15, permission=yes → ✓ You can watch the R-18 film.
age=15, permission=no → ✓ You can watch the PG-13 film.
age=10, permission=no → ✗ Sorry, no films available for age 10.
💡Parentheses in compound conditions: The starter code shows age >= 18 or (age >= 13 and permission == "yes"). Point out the parentheses — without them, or and and have different precedence (and binds tighter). When mixing operators, always use parentheses to make intent explicit.

Key vocabulary for this lesson
Logical operatorA keyword (and, or, not) that combines or inverts Boolean conditions.
Compound conditionTwo or more conditions joined by a logical operator into a single expression.
Truth tableA grid showing every possible True/False input combination and the resulting output.
Short-circuit evaluationPython stops evaluating a compound condition as soon as the result is certain.
Operator precedence (logic)not binds tightest, then and, then or. Use parentheses to override.

Teacher notes
🚨Most common mistake: Writing if age == 13 or 14 or 15 — students read this as "age equals 13 or 14 or 15" but Python reads it as "(age == 13) or (14) or (15)". Since non-zero integers are truthy, this condition is ALWAYS True. The fix: 13 <= age <= 15 or age == 13 or age == 14 or age == 15. Address this before Activity 1.
⚠️Common confusion: not A and B vs not (A and B). These are different — not has higher precedence than and so it binds to A alone. Activity 2 rows 4 and 5 address this directly. Spend the 3 minutes — it prevents a class of bugs that is very hard to spot.
⚠️Pacing note: The truth table activity (Activity 2) runs 8 minutes but is high-value. If time is short, assign rows 1–3 in class and rows 4–6 as homework — but do not skip the class discussion of rows 4 and 5. Those two rows teach more than the other four combined.
💡Building on Lesson 6: Point out that today's operators make Lesson 6's if-elif-else chains far more powerful. A grade checker needed 4 elif branches; with and, the range check 60 <= score < 75 collapses two conditions into one. Students should see this as a tool that simplifies code, not complicates it.
