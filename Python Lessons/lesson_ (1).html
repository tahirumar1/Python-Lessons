Learning objectives
  • Understand what Python is, where it's used, and why it's a great first language
  • Set up a working Python environment (installed or online via replit.com)
  • Write, run, and modify a first program using print()

Lesson timeline
0:00
Hook
Where is Python hiding? (5 min)
Open with a quick image slideshow or verbal quiz: "Can you guess which of these uses Python?" — Instagram, NASA, YouTube, Minecraft mods, your school website. Let students shout out guesses. Reveal answers. Goal: make Python feel relevant and already part of their world.
0:05
Teach
What is Python? (8 min)
Explain: Python is a programming language — a set of instructions a computer can follow. It was created by Guido van Rossum in 1991 and named after Monty Python. Cover: it's interpreted (runs line by line), readable (looks like English), free, and used everywhere. Draw the mental model: you write code → Python reads it → computer acts.
0:13
Setup
Environment setup (7 min)
Walk everyone through opening replit.com (no install needed, recommended for Day 1). Create a new Python repl. Point out: the code editor on the left, the run button at the top, and the output console on the right. If students have Python installed locally, show VS Code instead. Confirm everyone sees a blank editor before moving on.

Teach

Type each example on the projector, run it, discuss the output. Ask students to predict before you run. Introduce the vocabulary: function, argument, string, output, syntax. Show print() with text, numbers, and multiple arguments. Use think-aloud: narrate every keystroke and why.

Live demo — code examples

Type these on the projector one at a time. Let students predict the output before running.

Example 1 — The classic first line
main.py

Output
Hello, World!
Example 2 — Numbers (no quotes needed)
main.py
print(42)
print(3.14)
print(10 + 5)
Output
42
3.14
15
Example 3 — Multiple arguments (comma separates with a space)
main.py
print("My name is", "Aisha")
print("I am", 15, "years old")
Output
My name is Aisha
I am 15 years old
Example 4 — Blank lines with empty print()
main.py
print("=== My Profile ===")
print()
print("Name: Aisha")
print("Age: 15")
print("Favourite subject: Maths")
Output
=== My Profile ===

Name: Aisha
Age: 15
Favourite subject: Maths
Teaching tip: After Example 3, ask "What happens if we remove the quotes from Hello, World?" — let a student try it. This naturally leads into Activity 2.
Activity 1 — Hello World & beyond
Activity 1 · 10 minutes · Individual
Build your mini self-intro program
Students use print() to create a formatted personal introduction — at least 5 lines, including name, age, country/city, favourite thing, and one fun fact.
  1. Open a new Python file (or replit.com project)
  2. Start with a banner line using = or - characters
  3. Print your name, age, and location on separate lines
  4. Add your favourite hobby or subject
  5. End with a fun or silly fact about yourself
Starter code (give to students who are stuck)
main.py — starter

Activity 2 — Explore & break it
Activity 2 · 10 minutes · Pairs
Deliberately break the code, then fix it
Students introduce each error listed below, run the code, read the error message, fix it, and write one sentence explaining what the error meant. This builds confidence with error messages — they are helpful, not scary.
Start with this working code:
main.py — working version
print("Hello, World!")
print("Python is fun")
print(2024)
Now try each of these errors one at a time:
Error 1 — missing closing quoteSyntaxError
print("Hello, World!)
Error message: SyntaxError: EOL while scanning string literal — Python expects the string to close. Every " must have a matching ".
Error 2 — missing parenthesisSyntaxError
print "Hello, World!"
Error message: SyntaxError: Missing parentheses in call to 'print' — Python 3 requires parentheses around print arguments. This was valid in Python 2.
Error 3 — wrong capitalisationNameError
Print("Hello, World!")
Error message: NameError: name 'Print' is not defined — Python is case-sensitive. print ≠ Print ≠ PRINT.
Error 4 — unquoted textNameError
print(Hello)
Error message: NameError: name 'Hello' is not defined — without quotes, Python thinks Hello is a variable name, not text. Text must be wrapped in quotes.
Key message for students: Error messages are Python's way of helping you. Always read the last line of an error — it tells you exactly what went wrong and which line it's on.
Check for understanding — exit ticket
What happens if you remove the parentheses from print("Hello")?
A The program runs fine and prints Hello
B You get a SyntaxError because Python 3 requires parentheses
C The program prints nothing but doesn't crash
D Python automatically adds the parentheses for you
✓ Correct! In Python 3, print is a function and must be called with parentheses. Writing print "Hello" gives a SyntaxError.
Written part (students write in their notebooks): Write one print() statement of your own. Then write what error you would get if you removed the closing quote mark.
Key vocabulary for this lesson
ProgramA set of instructions that a computer executes in order.
FunctionA named action that does something. print() is a built-in function.
ArgumentThe value you pass into a function inside the parentheses.
StringText wrapped in quotes. e.g. "Hello" or 'World'.
SyntaxThe rules of how to write valid Python code.
SyntaxErrorPython couldn't understand your code — something was written incorrectly.

Teacher notes
Common mistake: Students using smart/curly quotes ("...") copied from a Word document. These cause SyntaxErrors. Always type directly in the code editor.
Common mistake: Students who leave a space before print — indentation errors will appear in later lessons. Mention that Python cares deeply about spaces at the start of a line.
Pacing: If setup takes longer than 10 minutes (common on day 1), skip Activity 2 and assign it as homework. Activity 1 is the priority — every student must successfully run their first program.
Differentiation: Fast finishers can try printing a pattern (triangle, box) using print() loops — but don't introduce loops yet. Just let them explore free-form.
Culture: Celebrate the first run loudly. The first "Hello, World!" is a milestone. Ask: "Who ran their first Python program today?" — have everyone raise a hand. It matters.
