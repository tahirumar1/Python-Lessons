🚨Lists are zero-indexed. The first element is at index 0, not 1. The last element is at index len(list) - 1. Accessing index len(list) or beyond raises an IndexError. This is the #1 list mistake — address it with live examples before Activity 1.
Essential list methods — display on projector
.append(item)
Adds item to the END of the list. Modifies in place, returns None.
fruits.append("fig") → ["apple","banana","cherry","fig"]
.remove(item)
Removes the FIRST occurrence of item. Raises ValueError if not found.
fruits.remove("banana") → ["apple","cherry","fig"]
.pop(index)
Removes and RETURNS the item at index. Default index is -1 (last item).
last = fruits.pop() → removes and returns "fig"
.insert(i, item)
Inserts item AT position i, shifting everything after it right.
fruits.insert(1, "blueberry") → at index 1
list[i] = value
Replaces the element at index i with a new value. Directly mutates the list.
fruits[0] = "avocado" → first item changes
len(list) · item in list
len() returns the count of items. in returns True/False for membership.
len(fruits) → 3   "apple" in fruits → True
💡append vs insert: append() is faster and more common — use it when order doesn't matter. insert() is for when position matters. Today focus on append, remove, and pop — the three students will use 90% of the time.
Live demo — code examples
Example 1 — Creating lists and basic indexing
main.py
# Create a list — square brackets, comma-separated
fruits = ["apple", "banana", "cherry", "date"]

# Positive indexing — starts at 0
print(fruits[0])    # "apple"   ← first
print(fruits[1])    # "banana"
print(fruits[3])    # "date"    ← last (index 3 = len-1)

# Negative indexing — counts from the end
print(fruits[-1])   # "date"    ← last
print(fruits[-2])   # "cherry"
print(fruits[-4])   # "apple"   ← same as [0]

# len() — number of items
print(len(fruits))  # 4

Example 2 — Mixed types and the IndexError
main.py
# Lists can hold any type — even mixed
profile = ["Aisha", 15, 1.65, True]
print(profile[0])   # "Aisha"
print(profile[1])   # 15

# ✗ IndexError — index out of range
nums = [10, 20, 30]
print(nums[3])      # IndexError! valid: 0,1,2 only

# ✓ Safe access — check length first
i = 3
if i < len(nums):
    print(nums[i])
else:
    print("Index out of range.")

Example 3 — Modifying a list
main.py
colours = ["red", "green", "blue"]
print(colours)         # ['red', 'green', 'blue']

# Reassign by index
colours[1] = "yellow"
print(colours)         # ['red', 'yellow', 'blue']

# append — add to end
colours.append("purple")
print(colours)         # ['red', 'yellow', 'blue', 'purple']

# remove — delete by value
colours.remove("red")
print(colours)         # ['yellow', 'blue', 'purple']

# pop — remove and return last item
removed = colours.pop()
print(removed)         # "purple"
print(colours)         # ['yellow', 'blue']

Example 4 — Iterating a list with for
main.py
scores = [85, 92, 67, 74, 90]

# Iterate directly — loop variable takes each value
for score in scores:
    print(f"Score: {score}")

# Iterate with index using range(len())
for i in range(len(scores)):
    print(f"  scores[{i}] = {scores[i]}")

# Find total and average
total = 0
for s in scores:
    total += s
print(f"Average: {total / len(scores):.1f}")
Output (partial)
Score: 85   Score: 92   Score: 67   Score: 74   Score: 90
scores[0] = 85   scores[1] = 92   ... scores[4] = 90
Average: 81.6
💡Direct iteration vs range(len()): Show both patterns side by side. Direct iteration (for item in list) is cleaner when you just need the values. range(len()) is needed when you also need the index. Students should know both — they'll use both constantly.
Example 5 — in, not in, and list queries
main.py
shopping = ["milk", "eggs", "bread", "butter"]

# Check membership
print("eggs" in shopping)          # True
print("cheese" in shopping)        # False
print("cheese" not in shopping)    # True

# Use in inside a condition
item = input("Check for item: ")
if item in shopping:
    print(f"✓ {item} is on the list.")
else:
    print(f"✗ {item} is NOT on the list.")

Activity 1 — Shopping list manager
Activity 1 · 13 minutes · Individual
Build a menu-driven shopping list app using a while loop and list methods
Students write a complete command-line app: a while True loop drives a menu, and each option calls a list method. This is the most complete program students have built — it uses every major concept from Weeks 1 through 3.
  1. Create an empty list: shopping = []
  2. Write a while True loop that prints a menu: (1) View list (2) Add item (3) Remove item (4) Quit
  3. Use if/elif/else to handle each menu choice
  4. View: print all items with their index using a for loop and f-strings
  5. Add: input() an item name, append() it, confirm with a message
  6. Remove: input() the item, check it's in the list before removing, give an error if not found
  7. Quit: break out of the while loop and print a farewell message
  8. Extension: add option (5) to clear the entire list and option (6) to count items
Starter code
main.py — starter
shopping = []

while True:
    print("\n=== Shopping List ===")
    print("1. View list")
    print("2. Add item")
    print("3. Remove item")
    print("4. Quit")

    choice = input("Choose (1-4): ")

    if choice == "1":
        if len(shopping) == 0:
            print("  (empty)")
        else:
            for i, item in enumerate(shopping):
                print(f"  {i+1}. {item}")

    elif choice == "2":
        item = input("  Item to add: ")
        shopping.append(item)
        print(f"  ✓ '{item}' added.")

    elif choice == "3":
        item = input("  Item to remove: ")
        if item in shopping:
            shopping.remove(item)
            print(f"  ✓ '{item}' removed.")
        else:
            print(f"  ✗ '{item}' not on the list.")

    elif choice == "4":
        print(f"List saved with {len(shopping)} items. Goodbye!")
        break
    else:
        print("  Please choose 1, 2, 3, or 4.")

Key vocabulary for this lesson
ListAn ordered, mutable collection of values. Created with square brackets: [1, 2, 3].
IndexThe position of an element in a list. Starts at 0. Negative indices count from the end.
MutableCan be changed after creation. Lists are mutable; strings and integers are not.
IndexErrorRaised when you try to access an index that doesn't exist — e.g. index 5 of a 4-item list.
append() vs pop()append adds to the end; pop removes from the end (or a given index) and returns the removed value.
in operatorTests membership: "apple" in fruits returns True/False. Works with lists and strings.

Teacher notes
🚨Zero-indexing — say it three times: The most common list mistake is list[1] when they mean the first element. Drill it physically: point to the list visualiser and ask "what is the index of apple?" every time a student hesitates. By lesson end it should be reflex.
⚠️Common mistake with remove(): Students who call .remove() without first checking in — if the item isn't in the list, Python raises a ValueError. The starter code for Activity 1 shows the safe pattern. Point it out explicitly.
⚠️Common confusion: Students who think list[i] = value and list.append(value) do the same thing. The first replaces an existing slot; the second adds a new slot at the end. Show both in the explorer and watch the list length change only with append.
💡enumerate() — introduce it but don't require it: The starter code uses enumerate(). Students who ask "what is that?" get a concise answer: "enumerate gives you both the index and the value at once." Students who don't notice it just use range(len()) — both approaches are valid today.
💡Activity 1 is the star of this lesson: The shopping list manager is the most complete program students have built so far — it combines while, if/elif, for, input, f-strings, and list methods. Celebrate when students get it working. It should feel powerful.
