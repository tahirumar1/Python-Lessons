Three iteration patterns — when to use each
① Direct — values only
for score in scores:
    print(score)
# Use when: you only need the value,
# not the position
② enumerate — index + value
for i, score in enumerate(scores):
    print(f"[{i}] {score}")
# Use when: you need both the index
# and the value (e.g. numbered lists)
③ range(len) — in-place modification
for i in range(len(scores)):
    scores[i] += 5   # add 5 to each
# Use when: you need to CHANGE the
# list elements themselves
④ Filter loop — collect matching items
passing = []
for s in scores:
    if s >= 50:
        passing.append(s)
# Use when: you need a subset of
# items meeting a condition
🎯Decision rule to put on the board: Values only → direct. Index + value → enumerate. Modify in place → range(len). Build a subset → filter loop. These four patterns cover the vast majority of real list tasks.

Live demo — code examples
Example 1 — Three iteration patterns compared
main.py
scores = [85, 92, 67, 74, 90]

# ① Direct — just the values
for s in scores:
    print(f"  score: {s}")

# ② enumerate — index + value
for i, s in enumerate(scores, 1):  # start=1
    print(f"  Student {i}: {s}")

# ③ range(len) — modify in place (+5 bonus)
for i in range(len(scores)):
    scores[i] += 5
print(f"  After bonus: {scores}")
Output
score: 85   score: 92   score: 67   score: 74   score: 90
Student 1: 85   Student 2: 92 ... Student 5: 90
After bonus: [90, 97, 72, 79, 95]
Example 2 — enumerate() start parameter
main.py
names = ["Aisha", "Omar", "Zara", "Bilal"]

# enumerate(list, start) — start counter at any number
for rank, name in enumerate(names, 1):
    print(f"  #{rank} — {name}")

# Without enumerate — the clunky way
for i in range(len(names)):
    print(f"  #{i+1} — {names[i]}")  # same, but more verbose
Both produce the same output
#1 — Aisha
#2 — Omar
#3 — Zara
#4 — Bilal

Example 4 — sort(), sorted(), reverse()
main.py
nums = [5, 2, 8, 1, 9, 3]

# sorted() — returns NEW list, original unchanged
asc  = sorted(nums)
desc = sorted(nums, reverse=True)
print(f"Original  : {nums}")   # unchanged!
print(f"Ascending : {asc}")
print(f"Descending: {desc}")

# .sort() — modifies IN PLACE, returns None
nums.sort()
print(f"After .sort(): {nums}")  # now sorted

# ✗ The trap — don't do this
trap = nums.sort()
print(trap)                    # None!!
Output
Original : [5, 2, 8, 1, 9, 3]
Ascending : [1, 2, 3, 5, 8, 9]
Descending: [9, 8, 5, 3, 2, 1]
After .sort(): [1, 2, 3, 5, 8, 9]
None

Example 6 — The filter loop pattern
main.py
scores  = [85, 42, 92, 38, 67, 51, 74]
passing = []   # empty list to collect results
failing = []

for s in scores:
    if s >= 50:
        passing.append(s)
    else:
        failing.append(s)

print(f"Passing ({len(passing)}): {sorted(passing, reverse=True)}")
print(f"Failing ({len(failing)}): {failing}")
Output
Passing (5): [92, 85, 74, 67, 51]
Failing (2): [42, 38]

Activity 2 — Word frequency counter
Activity 2 · 8 minutes · Individual
Split a sentence into words, then count each unique word using .count()
Students use .split() to turn a string into a list of words, then loop through unique words and use .count() to find how many times each one appears. Introduces the pattern of combining iteration with membership checking.
  1. Store a sentence: text = "the cat sat on the mat the cat sat"
  2. Split it into a list: words = text.split()
  3. Build a seen list to track which words you've already counted
  4. For each word in words: if word not in seen, print its count and add it to seen
  5. Extension: ask the user to enter their own sentence, then show the word with the highest frequency
Starter code
main.py — starter
text  = input("Enter a sentence: ").lower()
words = text.split()
seen  = []

print(f"\nWord frequencies ({len(words)} words total):")
for word in words:
    if word not in seen:
        count = words.count(word)
        print(f"  '{word}': {count}")
        seen.append(word)
Example output
Enter a sentence: the cat sat on the mat the cat sat

Word frequencies (9 words total):
'the': 3
'cat': 2
'sat': 2
'on': 1
'mat': 1
🔗Why .lower()? So "The" and "the" are treated as the same word. This is a real data-cleaning technique called normalisation — students are doing professional-level text processing without realising it.

Key vocabulary for this lesson
enumerate()Built-in that adds an index counter to any iterable. Returns (index, value) pairs each iteration.
sort() vs sorted().sort() mutates in place and returns None. sorted() returns a new list and leaves the original unchanged.
Filter loop patternCreating an empty list, then appending items that pass an if condition inside a for loop.
.split()String method that splits on whitespace (or a given separator) and returns a list of substrings.
.count(x)Returns how many times x appears in the list. Works on strings too: "hello".count("l") → 2.
Aggregate functionssum(), min(), max() — process an entire list and return a single summary value.

Teacher notes
🚨The sort() = None trap — demonstrate it live: Don't just warn students; run result = [3,1,2].sort(); print(result) on the projector and let them see None. The surprise is the lesson. Then immediately show sorted([3,1,2]) as the fix. Students who see the wrong output remember the rule.
⚠️Common mistake — modifying list while iterating: Students who try to remove() items from a list inside a for item in list loop will get unexpected behaviour — Python skips elements when the list changes size mid-loop. The safe pattern is to filter into a new list (the filter loop pattern from Example 6).
⚠️Common mistake — .index() with missing value: .index("x") raises ValueError if "x" is not in the list. Teach the safe pattern: check "x" in list before calling .index("x").
💡Lesson 14 preview — plant the seed: After showing the filter loop (Example 6), say: "You just wrote 4 lines. Tomorrow you'll write that in 1 line — a list comprehension. The logic is identical; the syntax is just compressed." Students who feel the verbosity today will be genuinely impressed tomorrow.
💡Activity 2 — .lower() is data cleaning: When students use .lower() before splitting, tell them this is called text normalisation and it's used in every real-world text processing system. The word frequency counter they just built is a primitive version of the algorithm that powers search engines and autocorrect.
