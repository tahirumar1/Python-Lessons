Anatomy of a for loop — display on projector
Every for loop has four components — unlike while, there is no separate update step
for i in range(1, 6): print(f"Number: {i}") # runs 5 times: i = 1,2,3,4,5
for — the keyword that starts the loop
i — the loop variable; takes each value from the sequence in turn
in range(1, 6) — the iterable; produces the sequence of values to loop over
indented body — runs once per value; no manual update needed — range handles it
🚨range(5) produces 0, 1, 2, 3, 4 — NOT 1, 2, 3, 4, 5. The stop value is always excluded. Students expect 1–5 and get 0–4. Say this three times, write it on the board, and show it live. Every single student will hit this off-by-one surprise at least once this week.
range() reference — all three forms
CallProducesCountUse when…
range(5)0, 1, 2, 3, 45 valuesYou want to repeat exactly N times (starts at 0)
range(1, 6)1, 2, 3, 4, 55 valuesYou need a specific start (e.g. 1-indexed)
range(0, 10, 2)0, 2, 4, 6, 85 valuesYou need every Nth value (even/odd/multiples)
range(10, 0, -1)10, 9, 8, … 110 valuesCounting down — step must be negative
range(1, 11, 3)1, 4, 7, 104 valuesStart 1, step 3, stop before 11
range(5, 5)(empty)0 valuesStart == stop → no iterations
💡Mental model for range(start, stop, step): Start at start. Keep going while the value is on the correct side of stop. Jump by step each time. Never include stop itself. This single rule explains every range() call.

for vs while — the decision rule
while — use when count is unknown
n = 5
while n > 0:
    print(n)
    n -= 1
print("Done")
Use while for: guessing games, "keep asking until valid", "run until user quits"
for — use when count is known
for n in range(5, 0, -1):
    print(n)
print("Done")
Use for for: tables, fixed repetitions, iterating over lists (Week 3 L12+), known sequences
💡Write this on the board and leave it there all week: "for = I know how many times. while = I'll know when to stop."

Example 3 — Nested for loops (multiplication table preview)
main.py
# 3×3 mini multiplication table
for row in range(1, 4):
    for col in range(1, 4):
        print(f"{row} × {col} = {row * col}")
    print()   # blank line after each row
Output
1 × 1 = 1   1 × 2 = 2   1 × 3 = 3

2 × 1 = 2   2 × 2 = 4   2 × 3 = 6

3 × 1 = 3   3 × 2 = 6   3 × 3 = 9
💡Nested loops — trace before running. Draw the two loops as concentric circles on the board: the outer loop moves the row, and for every row, the inner loop runs completely. Students need to see that the inner loop runs 3 full times for every single step of the outer loop.
Example 4 — break and continue in a for loop
main.py
# break: stop when we hit 5
for i in range(1, 10):
    if i == 5:
        break
    print(i)           # prints 1 2 3 4

# continue: skip multiples of 3
for i in range(1, 11):
    if i % 3 == 0:
        continue
    print(i)           # prints 1 2 4 5 7 8 10

Activity 2 — FizzBuzz
Activity 2 · 10 minutes · Individual
The classic programming challenge — using % from Lesson 4 inside a for loop
FizzBuzz is the most famous beginner programming exercise in the world. Students print numbers 1–30, but replace multiples of 3 with "Fizz", multiples of 5 with "Buzz", and multiples of both with "FizzBuzz". The key insight is checking the combined condition first.
  1. Write a for loop over range(1, 31)
  2. Check divisibility with %: n % 3 == 0 and n % 5 == 0
  3. Use if/elif/else — check FizzBuzz first, then Fizz, then Buzz, then the number
  4. Extension: generalise to any two user-supplied divisors and any range
Starter code
main.py — starter
for n in range(1, 31):
    if n % 3 == 0 and n % 5 == 0:  # ← check this FIRST
        print("FizzBuzz")
    elif n % 3 == 0:
        print("???")
    elif ???:
        print("Buzz")
    else:
        print(n)
🔗Why FizzBuzz first? If you check n % 3 == 0 first, then 15 prints "Fizz" and never reaches "FizzBuzz". The FizzBuzz condition must come before either individual condition — this is Lesson 6's ordering lesson applied to modulo. Make this connection explicit.

Key vocabulary for this lesson
for loopA loop that iterates over a sequence of values a fixed, known number of times.
range()A built-in function that generates a sequence of integers. Takes 1–3 arguments: stop, or start+stop, or start+stop+step.
Loop variableThe variable that takes each value from the sequence in turn (e.g. i in for i in range(10)).
IterableAny object Python can loop over. range() is one; strings and lists (next lesson) are others.
Nested loopA loop inside another loop. The inner loop completes fully for every single step of the outer loop.
Accumulator patternInitialise a variable (e.g. total = 0) before the loop; update it each iteration. Same pattern works in both for and while loops.

Teacher notes
🚨Off-by-one — the biggest for loop mistake: range(5) gives 0–4, not 1–5. Students who want 1–10 write range(10) and get 0–9. Show the interactive explorer on the projector and let them see numbers appear and disappear as they adjust start/stop. Visual feedback is faster than any explanation.
⚠️Common mistake in FizzBuzz: Checking n % 3 == 0 before the FizzBuzz condition — 15 then prints "Fizz" instead of "FizzBuzz". This is a deliberate trap. Let students discover it themselves during Activity 2 rather than pre-warning them — the discovery is the lesson.
⚠️Common confusion — modifying the loop variable: Students who write i += 1 inside a for loop think they're skipping ahead. They're not — range() replaces the value of i at the start of each iteration regardless. Use continue to skip, not manual increments.
💡Connecting to lists (Lesson 12): After the range explorer, say: "range() produces numbers. But for loops can iterate over anything with multiple values — including lists, which we meet tomorrow." This plants the seed and makes Lesson 12's hook land harder.
💡Pacing: Activity 1 (multiplication table) is quick for most students — 6–8 minutes. Activity 2 (FizzBuzz) is where students slow down and think. Budget the full 10 minutes for FizzBuzz and let the FizzBuzz playground be available for self-checking throughout.
