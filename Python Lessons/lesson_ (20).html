What this project brings together

This is the most structurally demanding project of the course. Every concept from Week 4 is applied: def and calling (L16), parameters and return values (L17), scope (L18), and multi-function program structure with call graphs (L19). The quiz game is also a direct test of Week 1–3 foundations — input, conditionals, loops, and lists all appear inside the functions.

🎯Project day goal: Every student leaves with a complete quiz game where every logical piece is a separate function — and they can explain what each one does. Circulate and ask "show me the function that checks the answer" rather than giving code.
Learning objectives
  • Design a program by identifying which logical pieces become functions before writing any code
  • Build at least 6 functions each with a clear single responsibility and docstring
  • Pass data between functions using parameters and return values — no unnecessary globals
  • Use a main() orchestrator that reads like a plain-English description of the program
  • Present code to a partner and explain every function's role and its connections

quiz_game.py — complete teacher solution
# ═══════════════════════════════════════════════════
#  QUIZ GAME — Week 4 Project                       
# ═══════════════════════════════════════════════════

# ═══ ZONE ① — Data ══════════════════════════════════

def get_questions():
    """Return a list of (question, answer, options) tuples."""
    return [
        ("What keyword defines a function?",
         "def",
         ["func", "define", "def", "function"]),
        ("What does a function return with no return statement?",
         "None",
         ["0", "False", "None", ""]),
        ("What is the output of: len([1, 2, 3])?",
         "3",
         ["1", "2", "3", "4"]),
        ("Which loop runs while a condition is True?",
         "while",
         ["for", "while", "loop", "repeat"]),
        ("What symbol starts a comment in Python?",
         "#",
         ["//", "--", "#", "/*"]),
    ]

# ═══ ZONE ① — Helper functions ═══════════════════════

def display_question(number, question, options):
    """Print a numbered question with lettered options."""
    print(f"\n  Q{number}: {question}")
    for i, opt in enumerate(options, 1):
        print(f"    {i}. {opt}")

def get_answer(num_options):
    """Prompt user for a valid answer (1 to num_options). Returns int."""
    while True:
        raw = input(f"  Your answer (1-{num_options}): ").strip()
        if raw.isdigit() and 1 <= int(raw) <= num_options:
            return int(raw)
        print(f"  ✗ Enter a number between 1 and {num_options}.")

def check_answer(chosen_option, options, correct_answer):
    """Return True if chosen option matches the correct answer."""
    return options[chosen_option - 1] == correct_answer

def show_feedback(correct, right_answer):
    """Print immediate right/wrong feedback after each question."""
    if correct:
        print("  ✓ Correct!")
    else:
        print(f"  ✗ Wrong. The answer was: {right_answer}")

def show_result(score, total):
    """Print final score and a performance message."""
    pct = round(score / total * 100)
    print(f"\n{'='*35}")
    print(f"  Final score: {score} / {total}  ({pct}%)")
    if pct == 100:
        print("  🎉 Perfect score!")
    elif pct >= 60:
        print("  👍 Good work! Keep practising.")
    else:
        print("  📚 Keep studying — you'll get there!")
    print(f"{'='*35}")

# ═══ ZONE ② — main() ════════════════════════════════

def main():
    """Run the quiz game."""
    questions = get_questions()
    score     = 0

    print("=== Python Quiz Game ===")
    print(f"Answer {len(questions)} questions. Good luck!\n")

    for i, (q_text, answer, options) in enumerate(questions, 1):
        display_question(i, q_text, options)
        chosen  = get_answer(len(options))
        correct = check_answer(chosen, options, answer)
        show_feedback(correct, answer)
        if correct:
            score += 1

    show_result(score, len(questions))

# ═══ ZONE ③ — Entry point ════════════════════════════

main()

Check for understanding — exit ticket
In the teacher solution, why is score updated in main() rather than inside check_answer()?
ABecause check_answer() can't access score — it is defined in a different scope
BBecause updating score inside check_answer() would require a global variable, which is slower
CBecause check_answer() has a single responsibility — deciding if the answer is right. Updating state belongs in the caller. This keeps each function focused and testable.
DBecause check_answer() runs before main() has defined the score variable

Week 4 key vocabulary — cumulative review
def / function calldef stores the recipe; calling with () executes it. A function does nothing until called (L16).
Parameter / argumentParameter is the placeholder in the definition; argument is the actual value at the call site (L17).
returnSends a value back to the caller. Without return, a function implicitly returns None (L17).
Local / global scopeLocal variables exist only inside their function. Global variables persist for the whole program (L18).
Single responsibilityEach function does exactly one thing. If you can't describe it in one sentence, it does too much (L19).
Three-zone layoutHelpers at top → main() in middle → entry call at bottom. The standard structure for Python programs (L19).

Teacher notes
🚨Most common structural problem: Students who put question data directly in main() — a long list of tuples right at the top of main(). Redirect immediately: "That data should live in get_questions(). main() should just call that function." The separation of data from logic is a professional pattern worth enforcing here.
⚠️Score as global — watch for it: Students who make score a global and use the global keyword in check_answer() are solving the problem but with bad practice. Accept it as working code, mark it down one point on structure, and say: "This works, but there's a cleaner way — check_answer() returns True/False, and main() decides whether to increment. Try refactoring."
⚠️Off-by-one in option indexing: Students who store options in a list indexed 0–3 but display them as 1–4, then access options[chosen] instead of options[chosen-1], will get wrong answers. The teacher solution shows the correct pattern. Watch for this in peer review.
💡Showcase selection: For the showcase, prioritise students who used an unusual design decision — e.g. a different data structure for questions, or an extra helper function not in the spec. Diversity of solutions teaches everyone more than seeing the same pattern four times.
💡Week 5 bridge: After revealing the teacher solution, show this one-liner change — replace the tuple list with a dict list and update display_question() to use q["question"] instead of q[0]. This is the perfect hook for Lesson 21. Students see immediately why dictionaries are more readable than positional tuples.
