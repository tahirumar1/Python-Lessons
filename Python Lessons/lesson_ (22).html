The three iteration methods — display on projector
.keys() — loop through keys
Use when you only need the keys — e.g. to check membership or build a list of field names.
for k in d.keys():
print(k)
→ "name" "age" "grade"
Note: for k in d also iterates keys — .keys() makes intent explicit.
.values() — loop through values
Use when you only need the values — e.g. to sum scores, find max, or check if a value exists.
for v in d.values():
print(v)
→ "Aisha" 15 "A"
Also: 92 in d.values() — checks values, not keys.
.items() — key + value together
Use when you need both — the most common iteration method. Unpacks into two variables.
for k, v in d.items():
print(k, v)
→ name Aisha / age 15 / grade A
Each iteration gives a (key, value) tuple that unpacks automatically.
💡Decision rule to put on the board: Need keys only → .keys() or plain for k in d. Need values only → .values(). Need both → .items(). In practice, .items() is used 80% of the time because you usually want both.

Example 1 — .keys(), .values(), .items() basics
main.py
student = {"name":"Aisha", "age":15, "grade":"A"}

# .keys() — view of all keys
print(student.keys())     # dict_keys(['name', 'age', 'grade'])
for k in student.keys():
    print(k)               # name  age  grade

# .values() — view of all values
print(student.values())   # dict_values(['Aisha', 15, 'A'])
for v in student.values():
    print(v)               # Aisha  15  A

# .items() — (key, value) pairs
print(student.items())    # dict_items([('name','Aisha'),...])
for k, v in student.items():
    print(f"  {k}: {v}")   # name: Aisha  age: 15  grade: A
Output
dict_keys(['name', 'age', 'grade'])
name
age
grade
dict_values(['Aisha', 15, 'A'])
Aisha
15
A
dict_items([('name', 'Aisha'), ('age', 15), ('grade', 'A')])
name: Aisha
age: 15
grade: A
💡dict_keys, dict_values, dict_items are view objects — not lists. They update automatically when the dict changes. You can convert them to lists with list(d.keys()) if you need a real list, but usually you just loop through them directly.
Example 2 — Practical use of .items(): grade report
main.py
scores = {"Aisha":92, "Omar":78, "Zara":95, "Bilal":65}

def get_grade(s):
    if s >= 90: return "A"
    if s >= 75: return "B"
    if s >= 60: return "C"
    return "F"

print(f"{'Name':10} {'Score':>6} {'Grade':>6}")
print("-" * 24)
for name, score in scores.items():
    grade = get_grade(score)
    print(f"{name:10} {score:>6} {grade:>6}")

# Use .values() for stats
all_scores = list(scores.values())
print(f"\nAverage: {sum(all_scores)/len(all_scores):.1f}")
print(f"Highest: {max(all_scores)}")
print(f"Lowest : {min(all_scores)}")

Example 3 — Checking values with .values()
main.py
inventory = {"apples":50, "bananas":0,
              "oranges":30, "mangoes":0}

# Check if a VALUE exists
print(0 in inventory.values())       # True — some items out of stock
print(50 in inventory.values())      # True

# Find items out of stock using .items()
print("\nOut of stock:")
for item, qty in inventory.items():
    if qty == 0:
        print(f"  ✗ {item}")

# Total stock using .values()
print(f"\nTotal items: {sum(inventory.values())}")

Example 4 — Word frequency counter with .items()
main.py
text  = "the cat sat on the mat the cat sat"
words = text.lower().split()
freq  = {}

# Build the frequency dict
for word in words:
    freq[word] = freq.get(word, 0) + 1

# Print using .items()
print("Word frequencies:")
for word, count in freq.items():
    print(f"  {word:10} × {count}")

# Most frequent word using .items()
top = max(freq.items(), key=lambda x: x[1])
print(f"\nMost common: '{top[0]}' ({top[1]} times)")
Output
Word frequencies:
the × 3
cat × 2
sat × 2
on × 1
mat × 1

Most common: 'the' (3 times)
🔗Connection to Lesson 13: Students built a word counter using .count() in a list loop — 5+ lines, O(n²) performance. The dict version is 2 lines and lightning fast. Ask: "Which would you use for a 10,000 word document?" The answer is always the dict.
Example 5 — .update() and .copy()
main.py
# .update() — merge another dict in
base   = {"name": "Aisha", "age": 15}
extras = {"city": "Karachi", "age": 16}  # age will overwrite
base.update(extras)
print(base)
# {'name':'Aisha','age':16,'city':'Karachi'}

# .copy() — independent shallow copy
original = {"score": 92, "grade": "A"}
backup   = original.copy()

original["score"] = 50   # modify original
print(original["score"])  # 50 — changed
print(backup["score"])    # 92 — unchanged!

# ✗ Without .copy() — alias, not copy
alias = original          # both names point to same dict
alias["score"] = 0
print(original["score"])  # 0 — original changed too!
Output
{'name': 'Aisha', 'age': 16, 'city': 'Karachi'}
50
92
0
🚨alias ≠ copy: b = a makes both names point to the same dict — changing b also changes a. b = a.copy() creates a genuinely independent copy. This is the same trap as list aliasing in Lesson 12 — connect to it explicitly.

Starter code
main.py — starter
text  = input("Enter a sentence: ").lower()
words = text.split()
freq  = {}

for word in words:
    freq[word] = freq.get(word, 0) + 1

print(f"\n{len(freq)} unique words:")
for word, count in freq.items():
    print(f"  {word:12} × {count}")

top = max(freq, key=freq.get)
print(f"\nMost common: '{top}' ({freq[top]} times)")
💡max(freq, key=freq.get): This finds the key with the largest value. max() iterates over the keys; key=freq.get tells it to compare by looking up each key's value. Students who are curious get a deeper explanation; others just use the pattern.

Preview — dict comprehensions (Lesson 24)

Just as list comprehensions compress filter loops into one line, dict comprehensions compress dict-building loops. Here's a taste — you'll build these properly in Lesson 24.

Building a "name → grade" dict from a scores dict — verbose vs comprehension
⚠ Verbose — 4 lines
grades = {}
for name, score in scores.items():
    grades[name] = get_grade(score)
print(grades)
✓ Dict comprehension — 1 line
grades = {name: get_grade(score)
           for name, score
           in scores.items()}
print(grades)
🔮Same result, one line. The structure is {key_expr: value_expr for k, v in d.items()}. Students who understood list comprehensions (Lesson 14) will find this very natural. Coming in Lesson 24.

Key vocabulary for this lesson
.keys()Returns a view object of all keys in the dictionary. Iterable. Use when you only need the key names.
.values()Returns a view object of all values. Use when you only need the values — summing, finding max, checking membership.
.items()Returns (key, value) tuples. The most common iteration method — use when you need both key and value.
Tuple unpacking in for loopfor k, v in d.items() — Python automatically unpacks each (key, value) pair into two variables.
.update(other)Merges another dict into this one. Existing keys are overwritten; new keys are added.
.copy()Returns an independent shallow copy. Changes to the copy do not affect the original — unlike assignment aliasing.

Teacher notes
🚨for k in d vs for k in d.keys(): Both iterate keys. The explicit .keys() makes intent clearer and is preferred in teaching. Some students will try for item in d.items() and print the tuple directly — that works, but show them the unpacking form for k, v in d.items() as the clean version.
⚠️dict_keys is not a list: Students who try to index d.keys()[0] get a TypeError — dict view objects don't support indexing. If they need the first key, they must convert: list(d.keys())[0]. Show this when it comes up rather than pre-emptively.
⚠️.update() silently overwrites: Students who call d.update(other) expecting a conflict warning will get none — existing keys are silently overwritten. Point out the interactive demo where "age" changes from 15 to 16 without any error. This is by design but surprises beginners.
💡Word frequency counter is the star activity: Students who did the Lesson 13 list-based word counter will feel the contrast immediately. The dict version is half the lines and vastly more readable. If time allows, show both side by side and ask which they'd rather maintain.
💡Lesson 23 bridge: Close by asking: "When .items() gave us (key, value) pairs in the loop — what type is each pair?" The answer is tuple. Students have been using tuples all lesson. Lesson 23 makes that explicit and shows when immutability is useful.
