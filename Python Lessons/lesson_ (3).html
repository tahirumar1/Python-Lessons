The big idea of this lesson
input() ALWAYS returns a string. If the user types 25, Python receives "25" — the text "25", not the number 25. You must convert with int() or float() before doing any arithmetic. This is the single most common error beginners make with input().

Example 3 — The type conversion problem and fix
main.py — broken first, then fixed
# ✗ BROKEN — input gives a string, can't add to number
age = input("How old are you? ")
print("Next year you will be", age + 1)
# → TypeError: can only concatenate str (not "int") to str

# ✓ FIXED — convert to int first
age = int(input("How old are you? "))
print("Next year you will be", age + 1)
Fixed output (user types 15)
How old are you? 15
Next year you will be 16
Example 4 — Three ways to combine strings (show all three, pick f-strings)
main.py
name = "Aisha"
age = 15

# Method 1 — concatenation (clunky, needs str conversion)
print("My name is " + name + " and I am " + str(age) + " years old.")

# Method 2 — comma-separated print (spaces auto-added)
print("My name is", name, "and I am", age, "years old.")

# Method 3 — f-string (best: readable, flexible, powerful)
print(f"My name is {name} and I am {age} years old.")
All three print the same thing
My name is Aisha and I am 15 years old.
My name is Aisha and I am 15 years old.
My name is Aisha and I am 15 years old.
Example 5 — Expressions inside f-strings
main.py
name = input("Your name: ")
age = int(input("Your age: "))
height = float(input("Your height in cm: "))

print(f"Hi {name}! You are {age} years old.")
print(f"Next year you will be {age + 1}.")
print(f"Your height in metres is {height / 100}.")
print(f"Your name has {len(name)} letters.")
Output (user enters: Aisha, 15, 165)
Your name: Aisha
Your age: 15
Your height in cm: 165
Hi Aisha! You are 15 years old.
Next year you will be 16.
Your height in metres is 1.65.
Your name has 5 letters.
Teaching tip: When showing Example 5 live, enter your own name and let students see {len(name)} compute the letter count automatically. The "wow" moment of expressions inside strings lands better with a real name than a made-up one.
Concatenation vs f-strings — display on board
Old way — string concatenation
"Hello " + name + "!"

Problems: needs str() for non-strings, hard to read when long, easy to miss spaces
New way — f-strings
f"Hello {name}!"

Benefits: variables go directly in braces, expressions work, reads like natural language
Activity 1 — Personal greeter

Build an interactive greeting program
Students write a program that asks the user several questions and responds with a personalised, multi-line message using f-strings. No hard-coded names — everything comes from input().

  2. Store age as an int using type conversion

Key vocabulary for this lesson
input()A built-in function that pauses the program and waits for the user to type something. Always returns a str.
Type conversionChanging a value from one type to another using int(), float(), or str().
f-stringA formatted string literal — prefix with f, embed variables or expressions inside {}.
PromptThe string inside input() that the user reads — tells them what to type.
TypeErrorAn error that occurs when you try to operate on incompatible types, e.g. adding a str and an int.
len()A built-in function that returns the number of characters in a string. Works inside f-strings too.

Teacher notes
Most common error: Forgetting to convert input() to int or float before doing arithmetic. When a student gets a TypeError, don't fix it for them — ask "what type does input() return?" and let them trace it.
Common mistake: Writing f"{name}" without the f prefix — it prints the literal text {name} instead of the value. Ask: "Is there an f before the opening quote?"
Common mistake: Using int() when the user might enter a decimal number. Use float() for any number that might have decimals — it handles whole numbers too.
Pacing note: If the class finishes both activities early, introduce f"{value:.2f}" for rounding floats to 2 decimal places in f-strings. This is previewed in the homework and lands well as a bonus.
Motivation moment: After Activity 1, pause and point out: "Your program from Lesson 1 always said the same thing. This one responds differently to every person who runs it. That's a real program." Let it land.
