Example 4 — Variable naming rules
main.py — valid vs invalid names
# ✓ Valid names
first_name = "Omar"
age2 = 16
is_logged_in = False
totalScore = 99   # works, but prefer snake_case

# ✗ Invalid names — these cause errors
# 2age = 16         → can't start with a number
# first name = "X"  → no spaces allowed

Teaching tip: After Example 3, ask "Did the variable name change?" (no) "Did the box move?" (no). The label stays; only what's inside changes. Revisit the whiteboard drawing.
Activity 1 — My profile variables
Activity 1 · 12 minutes · Individual
Create a variable for every piece of information about yourself
Students declare at least 6 variables covering different data types, then print each with a descriptive label. The constraint forces them to think about which type each piece of information should be.
  1. Create variables for: name (str), age (int), height in metres (float), favourite subject (str), is_morning_person (bool), and one variable of your own choice
  2. Print each variable on its own line with a label e.g. print("Name:", name)
  3. Add one line at the bottom that prints the type of each variable using type()
  4. Extension: try reassigning your age variable to age + 1 and print it — what happens?
Starter code
main.py — starter
# My Profile — fill in your own information!

name = "???"            # str  — your name
age = ???               # int  — your age
height = ???            # float — height in metres e.g. 1.65
favourite_subject = "???"  # str
is_morning_person = ???  # bool — True or False
my_variable = ???        # your choice!

# Print everything with labels
print("Name            :", name)
print("Age             :", age)
print("Height          :", height)
print("Favourite subject:", favourite_subject)
print("Morning person? :", is_morning_person)

# Check types
print(type(name), type(age), type(height), type(is_morning_person))
Example completed output
Name : Aisha Rahman
Age : 15
Height : 1.65
Favourite subject: Mathematics
Morning person? : False
<class 'str'> <class 'int'> <class 'float'> <class 'bool'>
Common mistake to watch for: Students writing age = "15" — they put quotes around a number by habit from the previous lesson. Ask: "Is age a piece of text, or a number you might do maths with?"

Check for understanding — exit ticket
Which data type does type(3.14) return?
Aint — because 3 and 14 are both whole numbers
Bstr — because it contains a dot which is a character
Cfloat — because it has a decimal point
Dbool — because 3.14 is a positive, truthy number
Written part: Create a variable called favourite_number and assign it a value. Then write one line that prints its type. What type did you get, and why?
Key vocabulary for this lesson
VariableA named container that stores a value in memory.
AssignmentUsing = to store a value in a variable. Not the same as "equals" in maths.
Data typeThe category of a value — what kind of thing it is (text, number, etc.).
snake_caseThe Python convention for naming variables: lowercase words joined by underscores.
ReassignmentGiving a variable a new value after it was already created.
Dynamic typingPython figures out the type automatically — you don't have to declare it.
Homework / self-study (before lesson 3)
Write a program that stores 8 facts about your favourite athlete, musician, or fictional character — using all four data types at least once. Print everything with neat labels.
Research question: What happens when you try print("Hello" + 5)? Run it, read the error, and write one sentence explaining what it means. We'll discuss in Lesson 3.
Look up: what are Python's reserved keywords? Write down 5 that you cannot use as variable names and why.
Teacher notes
Common mistake: Students write name = Aisha without quotes. Python looks for a variable called Aisha and crashes with NameError. Reinforce: text always needs quotes.
Common mistake: Confusion between = (assignment) and == (comparison). Don't introduce == today — just note "in Python, a single = means store, not equals. We'll meet double == in Lesson 5."
Common mistake: True and False written as true / false — Python is case-sensitive here. Only True and False (capitalised) are booleans.
Concept tip: Don't introduce type conversion (int(), str()) today — that's Lesson 3. If a student asks, say "great question, that's tomorrow's topic."
Pacing: The bonus challenge in Activity 2 ("10" + "5") surprises nearly every student. Budget 2–3 extra minutes if it sparks a good discussion — it's worth it. String concatenation with + previews Lesson 3 naturally.
