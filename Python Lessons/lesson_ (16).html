Learning objectives
  • Explain what a function is and why programs need them (DRY principle)
  • Define a function using def, a name, parentheses, a colon, and an indented body
  • Call a function by name and trace program execution into and out of it
  • Write a docstring to document what a function does
  • Understand that a function definition does nothing until the function is called
  • Refactor repeated code into a reusable function

Anatomy of a function — display on projector
Every function has five components — definitions do nothing; only calls run code
def greet(): """Prints a friendly greeting.""" print("Hello, World!") # Nothing has run yet — only defined greet() # ← this runs the body greet() # ← called again
def — keyword that says "I am defining a function"
greet — the function name; used to call it later. Snake_case, descriptive verb.
() — parentheses; empty for now (parameters come in Lesson 17)
indented body — everything indented under def is part of the function. Not run until called.
greet() — the call. This is what actually runs the body. Call it as many times as you like.
🚨Defining a function does NOT run it. Students constantly write a def block and wonder why nothing happens. The def just stores the recipe. You must call the function by name with () to execute it. This is the single most important point of Lesson 16.
DRY principle — Don't Repeat Yourself
✗ WET code — copy-pasted 3 times
# Banner for section 1
print("=" * 30)
print("  SECTION 1")
print("=" * 30)

# Banner for section 2
print("=" * 30)
print("  SECTION 2")
print("=" * 30)

# Banner for section 3
print("=" * 30)
print("  SECTION 3")
print("=" * 30)
9 lines. Want to change the width? Edit 6 places. Want to change = to -? Edit 6 places.
✓ DRY code — define once, call anywhere
def print_banner():
    print("=" * 30)

print_banner()
print("  SECTION 1")
print_banner()

print_banner()
print("  SECTION 2")
print_banner()

print_banner()
print("  SECTION 3")
print_banner()
Change the width in ONE place — all 6 calls update instantly.
🎯The DRY rule of thumb: If you find yourself writing the same (or nearly the same) code more than twice — stop. Give it a name. Make it a function. Every time you copy-paste code, you create a maintenance debt.
Live demo — code examples
Example 1 — Define once, call many times
main.py
def say_hello():
    print("Hello!")
    print("How are you today?")

print("Before first call")
say_hello()
print("Between calls")
say_hello()
print("After second call")
Output — predict order before running
Before first call
Hello!
How are you today?
Between calls
Hello!
How are you today?
After second call
💡Ask before running: "What prints first? What prints between the two calls? How many lines does the function body have?" Let students answer, then run. Trace execution with your cursor — leave the function, return to main code, leave again.
Example 2 — Definition before call; calling before defining
main.py
# ✓ Correct — define first, then call
def show_border():
    print("-" * 20)

show_border()   # works fine

# ✗ Wrong — call before define
wave()          # NameError: name 'wave' is not defined

def wave():
    print("👋")
Output
--------------------
NameError: name 'wave' is not defined
💡Rule to write on the board: Define before you call. Python reads files top to bottom. A function must exist in memory before you can use it. Convention: put all def blocks at the top of the file, calls at the bottom.
Example 3 — Docstrings and the DRY refactor
main.py
def print_banner():
    """Print a decorative section separator line."""
    print("=" * 40)

def show_welcome():
    """Display the app welcome screen."""
    print_banner()
    print("  Welcome to the Student Portal")
    print_banner()

def show_goodbye():
    """Display the app goodbye screen."""
    print_banner()
    print("  See you next time!")
    print_banner()

# --- Main program ---
show_welcome()
print("  (app runs here...)")
show_goodbye()

# Access the docstring
print(help(print_banner))
Output
========================================
Welcome to the Student Portal
========================================
(app runs here...)
========================================
See you next time!
========================================
Help on function print_banner:
print_banner() — Print a decorative section separator line.
💡Functions calling other functions: Point out that show_welcome() calls print_banner() inside its body. This is completely normal and very powerful. A function can call any other function that has already been defined above it.
Example 4 — The main() pattern
main.py — the professional structure
def draw_title():
    """Draw the app title block."""
    print("*" * 30)
    print("     MY APP")
    print("*" * 30)

def run_menu():
    """Show the main menu options."""
    print("1. Start")
    print("2. Help")
    print("3. Quit")

def main():
    """Entry point — orchestrates the whole app."""
    draw_title()
    run_menu()

main()   # the only line that runs at global level
🏗The main() pattern: Professional Python programs define all logic in functions and call only main() at the bottom. This makes code testable, readable, and easy to navigate. Start using it — students who adopt it now build better programs in every future lesson.

Activity 2 — DRY refactor challenge
Activity 2 · 10 minutes · Individual
Refactor a 30-line messy program into clean functions
Students receive a program that does three distinct things — each repeated or scattered across the file. They identify the repeating patterns, name them, extract them into functions with docstrings, and produce a clean version with a main() entry point.
  1. Read the starter code and identify three distinct logical blocks
  2. Give each block a descriptive function name (verb + noun, e.g. show_menu, print_score)
  3. Define each function above main() with a docstring
  4. Write main() that calls them in the right order
  5. Run both versions — confirm output is identical

# A messy program — refactor into 3 functions + main()

print("=" * 35)
print("     STUDENT GRADE REPORTER")
print("=" * 35)

scores = [85, 92, 67, 74, 90]
total  = 0
for s in scores:
    total += s
avg = total / len(scores)
print(f"  Scores  : {scores}")

print(f"  Highest : {max(scores)}")

print("=" * 35)

failing = [s for s in scores if s < 50]

Target structure after refactor
main.py — clean refactored version
def print_header(title):
    """Print a formatted section header."""
    print("=" * 35)
    print(f"     {title}")
    print("=" * 35)

def show_stats(scores):
    """Print score statistics for a list of scores."""
    avg = sum(scores) / len(scores)
    print(f"  Scores  : {scores}")
    print(f"  Average : {avg:.1f}")
    print(f"  Highest : {max(scores)}")
    print(f"  Lowest  : {min(scores)}")

def show_pass_fail(scores):
    """Print pass/fail breakdown for a list of scores."""
    passing = [s for s in scores if s >= 50]
    failing = [s for s in scores if s < 50]
    print(f"  Passing ({len(passing)}): {passing}")
    print(f"  Failing ({len(failing)}): {failing}")

def main():
    """Run the grade reporter."""
    scores = [85, 92, 67, 74, 90]
    print_header("STUDENT GRADE REPORTER")
    show_stats(scores)
    print_header("PASS / FAIL SUMMARY")
    show_pass_fail(scores)
    print_header("END OF REPORT")

main()
💡Notice: The refactored version is slightly longer in total lines, but far more readable — and the print_header() function now accepts a title argument (a preview of Lesson 17). Students who notice this are ready to think about parameters.

BThe function body needs a return statement to run
CThe function was defined but never called — a def block only stores the recipe; you must call the function with function_name() to run it

Key vocabulary for this lesson
FunctionA named, reusable block of code. Defined with def, executed only when called by name with ().
defThe keyword that begins a function definition. Followed by the name, parentheses, and a colon.
Function callInvoking a function by writing its name followed by (). This is what actually runs the body.
DocstringA string literal immediately after the def line, between """ triple quotes. Documents what the function does.
DRY principle"Don't Repeat Yourself" — if the same code appears more than twice, extract it into a function.
main()A conventional function name for the program entry point. Called once at the bottom of the file.

Teacher notes
🚨The #1 mistake of Lesson 16: Students who write a def block and then wonder why nothing happens. They defined the recipe but never made the tea. Every time this happens, ask: "Did you call it? Show me the line where you call the function." A missing function_name() line at the bottom is the cause 95% of the time.
⚠️Common mistake — calling before defining: Students who put the function call before the def block get a NameError. The fix is simple: move the def above the call. Reinforce the rule: Python reads top to bottom. Define before you use.
⚠️Common confusion — indentation: Students who put code outside the function body by mis-indenting think it's inside the function. Use the same coloured indentation visualiser mental model from Lesson 8. "Is this line indented under the def? If not, it's not part of the function."
💡Connect to Week 3 project: Start the lesson by opening the teacher solution from Lesson 15 and pointing to show_tasks(). Students have already seen a function in context. Say: "Today we learn to write these ourselves." This gives them a familiar anchor.
