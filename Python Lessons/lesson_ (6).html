The single most important rule of this lesson
🚨 Indentation is not optional in Python. The code inside an if block MUST be indented by exactly 4 spaces (or 1 tab — but be consistent). A missing or extra space causes an IndentationError that stops the entire program. This is the #1 source of errors on project day — hammer it today.

Live demo — code examples
Example 1 — Basic if / else
main.py
temperature = int(input("Enter temperature in °C: "))

if temperature > 30:
    print("It's hot! Wear sunscreen.")
else:
    print("Not too hot today.")
Two possible outputs
Enter temperature in °C: 35
It's hot! Wear sunscreen.

Enter temperature in °C: 22
Not too hot today.
Example 2 — if without else (valid!)
main.py
score = int(input("Enter your score: "))

if score == 100:
    print("Perfect score! 🎉")

print("Thanks for playing.")   # always runs — not indented
Score = 100 vs score = 85
Enter your score: 100
Perfect score! 🎉
Thanks for playing.

Enter your score: 85
Thanks for playing.
💡Point out: print("Thanks for playing.") is NOT indented — it runs regardless. Ask "what would happen if we indent it?" Students need to internalise that indentation = belongs inside the block.
Example 3 — if / elif / else chain
main.py
score = int(input("Enter score (0–100): "))

if score >= 90:
    print("Grade: A — Excellent!")
elif score >= 75:
    print("Grade: B — Good work.")
elif score >= 60:
    print("Grade: C — Keep going.")
elif score >= 50:
    print("Grade: D — Needs improvement.")
else:
    print("Grade: F — Please see the teacher.")
Three sample runs
Enter score: 93 → Grade: A — Excellent!
Enter score: 72 → Grade: C — Keep going.
Enter score: 41 → Grade: F — Please see the teacher.
Example 4 — Why order matters: broken vs fixed
main.py — BROKEN ordering
score = 95

# ✗ Wrong order — score >= 50 catches everything first!
if score >= 50:
    print("D or above")       # 95 hits this and STOPS
elif score >= 75:
    print("B or above")       # never reached for 95!
elif score >= 90:
    print("A")               # never reached either!

# ✓ Correct — always put most specific condition FIRST
if score >= 90:
    print("A")               # catches 90+ first
elif score >= 75:
    print("B")
elif score >= 50:
    print("D or C")
💡Run the broken version on the projector first. Let students be confused by the output, then ask "why did it print 'D or above' for 95?" The discovery that order matters sticks much better than being told upfront.
Activity 1 — Grade classifier
Activity 1 · 13 minutes · Individual
Build a complete grade classifier with personalised feedback
Students write an if-elif-else chain that takes a score, assigns a grade, and prints a personalised motivational message. Goes one step further than the demo — adds the student's name and a custom message per grade.
  1. Ask for the student's name and score using input() — convert score to int
  2. Write an if-elif-else chain for grades A (90+), B (75–89), C (60–74), D (50–59), F (below 50)
  3. Each branch prints the grade AND a personalised message using the student's name in an f-string
  4. After the chain (unindented), print the score back with :.0f formatting
  5. Extension: also print what score is needed to reach the next grade up
Starter code
main.py — starter
name  = input("Enter student name: ")
score = int(input("Enter score (0–100): "))

if score >= 90:
    print(f"{name}, you got an A! Outstanding work.")
elif score >= 75:
    print(f"{name}, you got a B. ???")    # add your message
elif score >= ???:                         # fill in the threshold
    print(f"{???}")
elif ???:
    print(f"{???}")
else:
    print(f"{name}, you got an F. Let's talk after class.")

print(f"Score recorded: {score}/100")
Example output
Enter student name: Zara
Enter score (0–100): 82
Zara, you got a B. Good work — aim for 90 next time!
Score recorded: 82/100
💡Extension hint: For "score needed to reach next grade", the logic is 75 - score for a B student needing an A, and so on. Students will need a different calculation in each branch — a great problem-solving exercise.
Activity 2 — Odd or even checker
Activity 2 · 10 minutes · Individual
Connect modulo (Lesson 4) to conditionals — is a number odd, even, or zero?
Students write a program that uses % inside an if condition — connecting arithmetic and conditionals for the first time. The extension adds divisibility checks (multiples of 3, 5, etc.) which previews the logical operators in Lesson 7.
  1. Ask for a whole number using int(input(...))
  2. Use if number == 0 to handle the zero case first
  3. Use elif number % 2 == 0 to detect even numbers
  4. Use else for odd numbers
  5. Extension: also check if the number is divisible by 3, 5, and 10 — print all that apply
Starter code
main.py — starter
number = int(input("Enter a whole number: "))

if number == 0:
    print("Zero is neither odd nor even.")
elif number % 2 == ???:   # what remainder means "even"?
    print(f"{number} is even.")
else:
    print(f"{???} is odd.")
Example outputs
Enter a whole number: 0 → Zero is neither odd nor even.
Enter a whole number: 12 → 12 is even.
Enter a whole number: 7 → 7 is odd.
🔗Connection moment: Before this activity, ask "which operator from Lesson 4 tells us the remainder?" Wait for students to call out %. Then say "today we use it inside a condition." This explicit connection builds cumulative confidence.

A student writes if score = 90:. What happens and why?
AIt works fine — Python treats = and == the same in a condition
BSyntaxError — = is assignment, not comparison. The condition needs ==
CIt runs but always evaluates to True regardless of score's value
DIndentationError — the if block is missing its indented body
Written part — trace this code on paper (do NOT run it):
x = 15   if x > 10: print("big")   elif x > 5: print("medium")   else: print("small")
Which line prints, and why? What would print if x = 7?
Key vocabulary for this lesson
Conditional statementCode that only runs when a condition is True. The if keyword starts one.
ConditionAn expression that evaluates to True or False. Written after if or elif, before the colon.
elifShort for "else if". Checked only when all previous conditions were False.
elseThe fallback branch — runs when every if and elif above it was False.
IndentationThe 4 spaces before code inside an if block. Python uses it to define scope — not optional.
BranchOne possible path through a conditional — the code block under a single if, elif, or else.

Teacher notes
🚨Most common error by far: if score = 90 instead of if score == 90. Say it out loud as a class: "double equals to compare". Repeat it every time a student hits this bug. By Week 8, they'll have it permanently.
⚠️Common error: Missing colon after the condition — if score >= 90 with no :. Causes SyntaxError. Teach the pattern as a unit: if [condition]: — the colon is part of the syntax, always.
⚠️Common misconception: Students who write multiple separate if blocks instead of elif — all conditions get checked independently. The homework question #3 addresses this directly. In Lesson 7, this distinction matters even more.
💡Connecting to prior learning: Activity 2 deliberately uses % from Lesson 4. Make this connection explicit before students start: "You already know the tool. Today you're using it in a new context." This builds confidence rather than treating each lesson as isolated.
💡Pacing note: Example 4 (broken ordering) is worth the extra 3 minutes even if it pushes the timeline. Students who discover the ordering bug themselves will never make it in their own code. A told rule is forgotten; a discovered rule is remembered.
