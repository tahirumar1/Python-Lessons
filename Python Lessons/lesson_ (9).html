Anatomy of a while loop — display on projector
Every while loop has exactly four components — all four are required
count = 5 # ① initialise while count > 0: # ② condition print(count) # ③ body count -= 1 # ④ update print("Blast off!") # after loop
① Initialise — set the variable that the condition depends on before the loop
② Condition — checked before every iteration; loop runs while True, exits when False
③ Body — the indented code that runs each iteration
④ Update — change the variable so the condition eventually becomes False. Forgetting this creates an infinite loop.
While loop flowchart — draw this on the board
The infinite loop — what it is and how to stop it
🚨 Infinite loops freeze your program — every beginner writes one. Here is how to survive it.
The broken loop — missing update
count = 5
while count > 0:
    print("Hello")
    # ✗ count never changes!
    # → prints "Hello" forever
How to stop a runaway loop:
Ctrl+CIn the terminal — sends a KeyboardInterrupt and stops the program immediately
■ StopThe red stop button in Replit, VS Code, or any online IDE
Prevention checklist: Before running a while loop, ask — (1) does the body change the variable the condition checks? (2) can the condition ever become False?

Example 1 — Basic countdown

count = 5

    print(f"{count}...")
    count -= 1
print("Blast off! 🚀")

4...
3...
2...
1...

Example 2 — Loop accumulator (summing numbers)
main.py
total = 0
count = 1
while count <= 5:
    total += count      # accumulate
    print(f"Added {count}, running total = {total}")
    count += 1
print(f"Final sum: {total}")
Output — trace this before running
Added 1, running total = 1
Added 2, running total = 3
Added 3, running total = 6
Added 4, running total = 10
Added 5, running total = 15
Final sum: 15
💡Accumulator pattern: initialise a variable to 0 before the loop, then add to it each iteration. This pattern — a variable that collects results across loop iterations — appears in almost every real program. Name it clearly: total, count, result.
Example 3 — break: exit early when a target is found
main.py
secret = 7
guess  = 0

while guess != secret:
    guess = int(input("Guess: "))
    if guess == secret:
        print("✓ Correct!")
        break           # exit immediately
    elif guess < secret:
        print("Too low.")
    else:
        print("Too high.")
Sample run
Guess: 3 → Too low.
Guess: 10 → Too high.
Guess: 7 → ✓ Correct!
Example 4 — continue: skip one iteration
main.py
n = 0
while n < 10:
    n += 1
    if n % 2 == 0:
        continue        # skip even numbers
    print(n)           # only odd numbers reach here
Output — only odd numbers
1 3 5 7 9
💡break vs continue: break exits the loop entirely. continue skips to the next check of the condition — the rest of the body is skipped but the loop keeps going. A useful mnemonic: "break = leave the room, continue = skip to the next question."
Example 5 — The "keep asking" validation pattern
main.py — improved safe divider from Lesson 8

number = int(raw)
print(f"Great! You entered: {number}")

Activity 1 — Countdown timer
Activity 1 · 10 minutes · Individual
Build a configurable countdown timer using a while loop
Students write a while loop that asks the user for a starting number, validates it is a positive integer, then counts down to zero and prints "Blast off!" Uses input validation from Lesson 8 combined with today's while loop.
  1. Ask the user for a starting number using input() — validate with .isdigit()
  2. If invalid, print an error and ask again using a while loop (keep-asking pattern)
  3. Once valid, convert to int and start a second while loop that counts down
  4. Each iteration: print the current count with an f-string, then decrement by 1
  5. After the loop: print "🚀 Blast off!" on its own line
  6. Extension: also print how many seconds the countdown took using a counter variable
Starter code

# Countdown Timer
raw = ""
while not raw.isdigit():
    raw = input("Start countdown from: ")
    if not raw.isdigit():
        print("  Enter a whole number.")

count = int(raw)

while count > ???:
    print(f"  {???}...")
    count ???= 1      # decrement

print("🚀 Blast off!")

4...
3...
2...
1...
🚀 Blast off!
Activity 2 — Guess the number game
Activity 2 · 12 minutes · Individual
Build a number guessing game — first use of import random
Students build a complete guessing game: the computer picks a random number, the player guesses until correct, getting "too high" or "too low" hints each time. Uses a while loop with break and introduces import random for the first time.
  1. Add import random at the top — use random.randint(1, 100) to pick a secret number
  2. Initialise a tries counter to 0 before the loop
  3. Write a while True: loop (infinite loop intentionally — break will exit it)
  4. Inside: ask for a guess, convert to int, increment tries counter
  5. Check: if correct → print congratulations with tries count and break
  6. If too low or too high → print the hint and loop again
  7. Extension: add a maximum of 7 tries before the game ends — use the tries counter
Starter code
main.py — starter
import random

secret = random.randint(1, 100)
tries  = 0

print("I'm thinking of a number between 1 and 100.")

while True:                          # loop forever until break
    guess = int(input("Your guess: "))
    tries += 1

    if guess == secret:
        print(f"🎉 Correct! You got it in {tries} tries.")
        break
    elif guess < secret:
        print("Too low — try higher.")
    else:
        print("Too high — try lower.")
Sample output
Your guess: 50 → Too high — try lower.
Your guess: 25 → Too low — try higher.
Your guess: 37 → Too high — try lower.
Your guess: 31 → 🎉 Correct! You got it in 4 tries.
🔗while True with break is a valid pattern. It reads as "loop forever, but break when done." It is cleaner than a flag variable like running = True in many simple games. Point this out — students will see while True in real Python code and should recognise it.
💡Week 2 project preview: Tomorrow's project (Lesson 10) expands this game — adding a play-again loop and a best-score tracker. Students who finish this activity have essentially built the core of the project already. Fast finishers should start the extension (7-try limit) now.
Check for understanding — exit ticket
What is the difference between break and continue inside a while loop?
Abreak skips the current iteration; continue exits the loop entirely
Bbreak exits the loop entirely; continue skips the rest of the current iteration and goes back to check the condition
CBoth do the same thing — they both stop the loop from running
Dbreak pauses the loop; continue restarts it from the beginning
Written part — trace this loop on paper and write every line it prints:
n = 1   while n <= 6:   if n == 4: n += 1   continue   print(n)   n += 1
(Watch out — the update inside the if changes things!)
Key vocabulary for this lesson
while loopRepeats a block of code as long as a condition is True. Stops when the condition becomes False.
Infinite loopA while loop whose condition never becomes False — it runs forever. Caused by forgetting to update the condition variable.
breakImmediately exits the loop, regardless of the condition. Execution jumps to the first line after the loop.
continueSkips the rest of the current iteration and jumps back to the condition check.
AccumulatorA variable initialised before a loop and updated each iteration to collect a running total or result.
while TrueA loop with a condition that is always True — relies entirely on break to exit. Common pattern for menu-driven programs and games.

Teacher notes
🚨Run an infinite loop on purpose. Students must experience it — not just hear about it. Let it print 10–15 lines, then press Ctrl+C. Say "that would have run forever." The visceral experience of stopping a runaway program is more memorable than any explanation.
⚠️Most common mistake: Forgetting to update the loop variable — leading to an infinite loop. Ask students to show you their update line before they run any while loop. A 5-second check prevents 10 minutes of frozen editors.
⚠️Common confusion: while True looks wrong to beginners — "won't that loop forever?" Explain: yes, by design. The break is the real exit condition. It's a deliberate pattern, not a bug. Show Examples 3 and the guessing game side by side.
💡Pacing note: Activity 2 (guessing game) is the higher-value activity. If time is short, reduce Activity 1 to just the countdown without validation — students can add validation in homework. Do not cut Activity 2; it is the Week 2 project prototype.
💡import random preview: Students will be curious about random.randint(). Keep it brief — "random is a module (a toolbox of functions) that Python includes for free. We'll explore modules properly in Week 6. For now, just know that random.randint(1, 100) picks a random whole number between 1 and 100 inclusive."
