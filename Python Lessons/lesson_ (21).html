Anatomy of a dictionary — display on projector
A dictionary maps keys to values — keys must be unique, values can be anything
student = { "name": "Aisha", "age": 15, "grade": "A", "score": 92.5 }
{ } — curly braces wrap the whole dictionary. Different from lists (square brackets) and tuples (parentheses).
"name" — the key. Usually a string. Must be unique within the dict. Immutable types only (strings, ints, tuples).
: — separates each key from its value. Read it as "maps to".
"Aisha" — the value. Any Python type: string, int, float, bool, list, even another dict.
Pairs separated by commas. Trailing comma on last pair is fine and often used.
List vs dictionary — when to use each
⚠ List — positional, unnamed
# What is index 2? You must remember.
student = ["Aisha", 15, "A", 92.5]
print(student[0])   # "Aisha" — but why 0?
print(student[3])   # 92.5 — is this score?
Position is fragile — insert an item and all indices shift. Reader must memorise what each position means.
✓ Dictionary — named, self-documenting
# The key tells you exactly what you're getting.
student = {"name":"Aisha", "age":15,
            "grade":"A", "score":92.5}
print(student["name"])   # "Aisha" — obvious
print(student["score"])  # 92.5 — no guessing
Keys are meaningful names — code is readable without comments. Order doesn't matter for access.
🎯Rule of thumb: Use a list for an ordered collection of similar items (scores, names, tasks). Use a dictionary for a collection of named properties belonging to one thing (a student profile, a quiz question, a product).

Example 1 — Creating and accessing a dictionary
main.py
# Create a dictionary
student = {
    "name":   "Aisha",
    "age":    15,
    "grade":  "A",
    "score":  92.5,
    "passed": True
}

# Access values with the key
print(student["name"])    # "Aisha"
print(student["score"])   # 92.5
print(student["passed"])  # True

# Use in f-strings
print(f"{student['name']} scored {student['score']} — grade {student['grade']}")

# KeyError — key does not exist
print(student["email"])   # KeyError!

Example 2 — Safe access with .get()
main.py
student = {"name": "Aisha", "age": 15}

# .get() returns None if key missing — no crash
print(student.get("name"))    # "Aisha"
print(student.get("email"))   # None

# .get() with a default value
print(student.get("email", "not set"))  # "not set"
print(student.get("age", 0))             # 15 (key exists, default ignored)
Output
Aisha
None
not set
15
🚨d[key] vs d.get(key): Use d[key] when you are certain the key exists — it raises KeyError clearly if it doesn't, which helps debugging. Use d.get(key) when the key might be absent and you want a safe fallback. Mixing these up is the #1 dictionary bug.
Example 3 — Adding and updating entries
main.py
profile = {"name": "Omar", "city": "Lahore"}
print(profile)   # {'name': 'Omar', 'city': 'Lahore'}

# Add a NEW key — same syntax as update
profile["age"] = 17
print(profile)   # {'name': 'Omar', 'city': 'Lahore', 'age': 17}

# UPDATE existing key — overwrites silently
profile["city"] = "Karachi"
print(profile)   # {'name': 'Omar', 'city': 'Karachi', 'age': 17}

# Increment a numeric value
profile["age"] += 1
print(profile["age"])   # 18
Output
{'name': 'Omar', 'city': 'Lahore'}
{'name': 'Omar', 'city': 'Lahore', 'age': 17}
{'name': 'Omar', 'city': 'Karachi', 'age': 17}
18
💡Add vs update — same syntax, different effect: Python doesn't care whether the key already exists. d[key] = value adds a new pair if the key is new, or silently overwrites the value if the key exists. Students who expect a warning when overwriting are often surprised — point this out explicitly.
Example 4 — Deleting entries
main.py
car = {"make": "Toyota", "model": "Corolla",
        "year": 2020,  "colour": "white"}

# del — removes key, returns nothing
del car["colour"]
print(car)   # {'make':'Toyota','model':'Corolla','year':2020}

# .pop() — removes AND returns the value
yr = car.pop("year")
print(yr)    # 2020
print(car)   # {'make': 'Toyota', 'model': 'Corolla'}

# .pop() with default — safe, no KeyError
col = car.pop("colour", "unknown")
print(col)   # "unknown" — key was already gone
Output
{'make': 'Toyota', 'model': 'Corolla', 'year': 2020}
2020
{'make': 'Toyota', 'model': 'Corolla'}
unknown
Example 5 — Membership, len(), and empty dict
main.py
scores = {"Aisha": 92, "Omar": 78, "Zara": 85}

# in checks KEYS only
print("Aisha" in scores)    # True
print("Bilal" in scores)    # False
print(92 in scores)         # False — 92 is a VALUE, not a key

# len() — number of key-value pairs
print(len(scores))           # 3

# Safe access pattern — check before access
name = "Bilal"
if name in scores:
    print(f"{name}: {scores[name]}")
else:
    print(f"{name} not found")

# Empty dict — create now, fill later
inventory = {}
inventory["apples"] = 50
inventory["bananas"] = 30
print(inventory)             # {'apples': 50, 'bananas': 30}
Output
True
False
False
3
Bilal not found
{'apples': 50, 'bananas': 30}
💡in checks keys, not values. Students who expect 92 in scores to be True because 92 is in the dict will be surprised. in on a dict only checks the keys. To check values, use 92 in scores.values() — preview for Lesson 22.

Preview — using a dict to count word frequency

In Lesson 22 you'll use dict methods like .keys(), .values(), and .items() to loop through dictionaries. Here's a taste: a word counter built with a dict. Type any sentence and see how words map to counts — this is the pattern you'll write yourself next lesson.

Word frequency counter — powered by a dictionary behind the scenes
Dictionary result — keys are words, values are counts:
The Python code that builds this (preview — uses .get() you learned today):
freq = {}
for word in input().lower().split():
    freq[word] = freq.get(word, 0) + 1
# freq.get(word, 0) returns current count or 0 if new word
# +1 increments it — then stores back into freq[word]
🔮This one-liner uses .get(): freq.get(word, 0) + 1 — if the word is new, .get() returns 0, then adds 1. If it exists, .get() returns its current count, then adds 1. This is one of the most common and elegant dict patterns in Python. Students who understand .get() from today will find this natural tomorrow.

Key vocabulary for this lesson
DictionaryA mutable collection of key-value pairs created with {}. Keys must be unique and immutable. Values can be any type.
KeyThe name used to look up a value in a dictionary. Usually a string. Must be unique within the dict.
ValueThe data associated with a key. Can be any Python type — string, int, list, even another dictionary.
KeyErrorRaised when you try to access a key that doesn't exist using d[key]. Prevented by .get() or in check.
.get(key, default)Safe dictionary access. Returns the value if key exists, otherwise returns the default (or None if no default given).
.pop(key)Removes a key-value pair and returns the value. Unlike del, the value is accessible after removal.

Teacher notes
🚨KeyError is the #1 dictionary bug: Students who write d["email"] when the key might not exist will crash their programs. Run a live demo of the crash on the projector — then immediately show the .get() fix. The visual of the red error message is more memorable than any explanation.
⚠️Common confusion — in checks keys not values: Students who expect 92 in scores to return True because 92 is stored in the dict are confused by this. Show it live. Then say: "If you want to check values, that's 92 in scores.values() — and we'll cover that in Lesson 22."
⚠️Silent overwrite on duplicate keys: d["name"] = "Aisha" followed by d["name"] = "Omar" silently replaces Aisha with Omar — no error, no warning. Students who expect a duplicate-key error are often surprised. This becomes a real bug in the contact book homework if they add the same name twice.
