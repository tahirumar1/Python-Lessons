Anatomy — parameters in the definition, arguments in the call
Parameters are placeholders. Arguments are the actual values. The mapping happens at call time.
# DEFINITION — parameter is placeholder def greet(name): print(f"Hello, {name}!") # CALLS — argument is the real value greet("Aisha") # name = "Aisha" greet("Omar") # name = "Omar" greet("Zara") # name = "Zara"
name — the parameter. A local variable inside the function. Exists only while the function runs.
"Aisha" — the argument. The actual value passed at the call site. Bound to the parameter name.
Each call passes a different argument → same function, different behaviour. This is the whole point of parameters.
Number of arguments must match number of parameters (unless defaults are used). Mismatch → TypeError.
Parameter vs argument — the distinction that confuses everyone
TermWhere it appearsWhat it isExample
ParameterInside the def lineA placeholder variable — gets its value when the function is calleddef greet(name): — name is the parameter
ArgumentInside the function call ()The actual value passed in — fills the parameter's placeholdergreet("Aisha") — "Aisha" is the argument
Default parameterInside the def line with =A parameter with a fallback value — used when no argument is passeddef greet(name="friend"):

💡In everyday speech, most people say "parameter" and "argument" interchangeably — and that is fine. The distinction matters for precision: parameter is in the definition, argument is in the call. Don't spend more than 2 minutes on the naming; spend the time on how they work.
Return values — the function sends something back
print-only function vs return function — a crucial difference
✗ print only — result disappears
def add(a, b):
    print(a + b)   # shows it

result = add(3, 4)
print(result)      # None!
print(result * 2) # TypeError!
The value is printed inside the function, then lost. The caller can't use it.
✓ with return — caller receives the value
def add(a, b):
    return a + b   # hands it back

result = add(3, 4)
print(result)      # 7
print(result * 2) # 14 ✓
The value is returned to the caller. It can be stored, printed, or used in further calculations.
🚨A function without return implicitly returns None. Every Python function returns something. If you don't write a return statement, it returns None. Students who store the result of a print-only function are always surprised to find None. Show this live — it's the most common function mistake after Lesson 16.

Example 1 — Single parameter
main.py
def greet(name):
    """Print a personalised greeting."""
    print(f"Hello, {name}! Welcome to Python class.")

greet("Aisha")
greet("Omar")
greet("Zara")

# What happens with the wrong number of arguments?
greet()       # TypeError: missing 1 required argument
greet("A", "B") # TypeError: takes 1, given 2
Output (first three calls)
Hello, Aisha! Welcome to Python class.
Hello, Omar! Welcome to Python class.

Example 2 — Multiple parameters
main.py
def describe_pet(name, animal, age):
    """Describe a pet with name, type and age."""
    print(f"{name} is a {age}-year-old {animal}.")

describe_pet("Fluffy", "cat", 3)
describe_pet("Rex", "dog", 7)
describe_pet("Tweety", "bird", 2)

# Keyword arguments — order doesn't matter!
describe_pet(age=5, name="Goldie", animal="fish")
Output
Fluffy is a 3-year-old cat.
Rex is a 7-year-old dog.
Tweety is a 2-year-old bird.
Goldie is a 5-year-old fish.
💡Keyword arguments bonus: The last call uses keyword arguments — you name each argument explicitly so order doesn't matter. This is very readable and common in real Python code. Students don't need to master it today, but seeing it removes confusion when they encounter it.
Example 3 — return values
main.py
def celsius_to_fahrenheit(c):
    """Convert Celsius to Fahrenheit."""
    return c * 9 / 5 + 32

# Store and use the return value
boiling = celsius_to_fahrenheit(100)
body    = celsius_to_fahrenheit(37)
freezing = celsius_to_fahrenheit(0)

print(f"Boiling : {boiling}°F")
print(f"Body    : {body:.1f}°F")
print(f"Freezing: {freezing}°F")

# Use return value directly in expressions
print(celsius_to_fahrenheit(20) > 68)   # True
print(round(celsius_to_fahrenheit(37), 1))   # 98.6
Output
Boiling : 212.0°F
Body : 98.6°F
Freezing: 32.0°F
True
98.6
Example 4 — None trap: print vs return
main.py — the trap and the fix
# ✗ print only — result is lost
def add_bad(a, b):
    print(a + b)

x = add_bad(3, 4)  # prints 7, but x = None
print(x)            # None

# ✓ return — result is usable
def add_good(a, b):
    return a + b

x = add_good(3, 4)  # x = 7
print(x)             # 7
print(x * 2)         # 14

# Rule: print to display · return to compute
Output
7
None
7
14
Example 5 — Default parameters
main.py
def calculate_price(price, discount=10):
    """Return price after discount. Default discount is 10%."""
    return price * (1 - discount / 100)

# Using default discount (10%)
print(calculate_price(500))         # 450.0

# Overriding the default
print(calculate_price(500, 20))     # 400.0
print(calculate_price(500, 0))      # 500.0 — no discount

# ✗ Wrong order — defaults must come LAST
def bad_func(discount=10, price):  # SyntaxError!
    pass
Output
450.0
400.0
500.0
💡Default parameter rule: Parameters without defaults must come first. Python needs to know which argument fills which parameter in positional calls. If a required parameter came after an optional one, Python couldn't figure out which was which. Run the bad_func example live to show the SyntaxError.

Activity 2 — Discount calculator
Activity 2 · 10 minutes · Individual
Build calculate_price() with a default parameter, then chain return values
Students write a price calculator with a default discount, then use its return value as the input to a tax calculator. This chaining of return values — using the output of one function as the argument to another — is a key programming pattern.
  1. Write calculate_price(price, discount=10) — returns price after discount
  2. Write add_tax(price, tax_rate=17) — returns price with tax added
  3. Call calculate_price(1000) and store the result
  4. Pass that result into add_tax() — chain the two functions
  5. Print the original, discounted, and final prices using f-strings
  6. Extension: write a final_price(original, discount=10, tax=17) that calls both functions internally

Key vocabulary for this lesson
ParameterA variable in the function definition that acts as a placeholder for the value that will be passed in.
ArgumentThe actual value passed to a function when it is called. Fills the corresponding parameter.
returnSends a value from inside the function back to the caller. Immediately exits the function.
Default parameterA parameter with a fallback value (discount=10). Used when the caller doesn't provide an argument. Must come after non-default parameters.
NoneThe value Python returns when no return statement is present. Not 0, not "", not False — it means "no value".
Keyword argumentPassing an argument by parameter name: greet(name="Aisha"). Makes the call self-documenting and order-independent.

Teacher notes
🚨The None trap — demonstrate it live: Run x = print("hello") and then print(x). Students see hello followed by None. Then run x = add_bad(3, 4) where add_bad only prints. Same result. The lesson: print displays, return gives back. You need return to use the value elsewhere.
⚠️Common mistake — return inside a loop: Students who write a loop inside a function and put return inside the loop exit after the first iteration. The return should usually come after the loop. This comes up naturally in Activity 2 extension — watch for it.
⚠️Common mistake — default parameters in wrong order: def f(discount=10, price): causes a SyntaxError. Show it live. The rule is: required parameters first, defaults last. Python can't resolve which value goes where otherwise.
💡print vs return — a mantra to repeat: "print to display, return to compute." A function that prints is like a one-way door — the value goes out but nothing comes back. A function that returns is like a two-way exchange. Write it on the board and leave it there.
